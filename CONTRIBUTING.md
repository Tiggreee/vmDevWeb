# Contributing

Thanks for contributing to vmDevWeb.

## Branching

- Base branch: `main`
- Branch naming: `feat/<scope>`, `fix/<scope>`, `chore/<scope>`

## Commit Style

Use Conventional Commits when possible:
- `feat:` new behavior
- `fix:` bug fix
- `docs:` documentation
- `chore:` maintenance

## Pull Requests

Before opening a PR:
- Keep changes focused and small
- Update docs when behavior changes
- Ensure CI passes

PRs should include:
- What changed
- Why it changed
- Risks and rollback notes

## Code of Collaboration

Be direct, respectful, and technical.
